
Механизм работы:Получение идентификатора и ключей: Рест-сервис принимает идентификатор файла и ключи для аутентификации.Запрос на создание сессии: Используя полученные идентификатор и ключи, рест-сервис отправляет запрос на API eef.gov.kz для создания сессии.Получение сигнатуры по идентификатору файла: После успешного создания сессии рест-сервис отправляет запрос на получение сигнатуры файла по указанному идентификатору.Подписание сигнатуры: Полученная сигнатура подписывается локально на вашем сервере с использованием ключей ЭЦП GOST или RSA.Получение сигнатуры действия: Подписанная сигнатура отправляется обратно на API eef.gov.kz для получения сигнатуры действия.Изменение статуса файла: Полученная сигнатура действия отправляется на API eef.gov.kz для изменения статуса файла.Обработка ответа: Рест-сервис ожидает ответа от API eef.gov.kz и обрабатывает его в соответствии с логикой вашего приложения.Таким образом, ваш рест-сервис осуществляет безопасное подписание файлов с использованием ЭЦП GOST или RSA и взаимодействует с API eef.gov.kz для выполнения операций, таких как изменение статуса файла.
