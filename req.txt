Дополнение к функции JavaScript
В вашей функции OnComboChanged вы уже меняете видимость ASPxMemo. Чтобы сделать так, чтобы элемент <strong> также появлялся и исчезал, добавьте к этой функции изменение свойства display соответствующего элемента. Предположим, что у вашего элемента <strong> есть идентификатор strongElement.

javascript
Copy code
function OnComboChanged(combo) {
    var showComponents = combo.GetValue() === "ShowMemo";
    var memo = ASPxClientControl.GetControlCollection().GetByName("memo");
    var strongElement = document.getElementById("strongElement"); // Предполагаем, что таковой ID

    if(memo && strongElement) {
        memo.SetVisible(showComponents);
        strongElement.style.display = showComponents ? "block" : "none"; // Показать или скрыть <strong>
        // Если ASPxMemo видим, включаем валидацию
        var validator = document.getElementById("<%= RequiredFieldValidatorMemo.ClientID %>");
        ValidatorEnable(validator, showComponents);
    }
}

Обязательность ASPxMemo
Для того чтобы сделать ASPxMemo обязательным только когда он видим, следуйте инструкциям, которые были предоставлены ранее, добавляя валидацию на клиентской стороне и управляя её активацией в той же функции OnComboChanged, что и видимость элементов.

Таким образом, вы получите динамическое управление видимостью элементов и валидацией ASPxMemo на основе выбора пользователя в ASPxComboBox, полностью на стороне клиента без необходимости серверного кода для этой конкретной логики.
