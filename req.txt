Для Oracle запрос с оператором `CONNECT BY` должен работать корректно. Убедитесь, что структура таблицы и данные в полях соответствуют иерархическому запросу. Вот ещё раз пример с некоторыми уточнениями:

```sql
SELECT LEVEL, employee_id, first_name, manager_id
FROM employees
START WITH manager_id IS NULL -- Указывает на корневой элемент (где руководитель отсутствует)
CONNECT BY PRIOR employee_id = manager_id; -- Определяет связь между сотрудником и его руководителем
```

**Объяснение:**
- `LEVEL` — специальная функция в Oracle, которая возвращает уровень иерархии (1 для корневых элементов).
- `START WITH manager_id IS NULL` — выбирает сотрудников, у которых нет руководителя (верхний уровень иерархии).
- `CONNECT BY PRIOR employee_id = manager_id` — находит сотрудников, чьи `manager_id` соответствуют `employee_id` их начальника.

Проверьте следующее:
1. Поле `manager_id` действительно содержит `NULL` для сотрудников, у которых нет руководителя.
2. Поля `employee_id` и `manager_id` логически взаимосвязаны: `manager_id` одного сотрудника должен ссылаться на `employee_id` другого.
3. Структура данных таблицы корректна для иерархического запроса.

Если ошибка остаётся, пришлите текст ошибки, чтобы можно было точнее определить проблему.
