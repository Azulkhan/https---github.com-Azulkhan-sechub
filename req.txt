В PL/SQL Developer вы можете сравнить изменения в ваших SQL скриптах, пакетах, триггерах и других объектах базы данных с оригинальными версиями. Для этого выполните следующие шаги:

1. **Используйте инструмент сравнения**: В PL/SQL Developer есть встроенная возможность сравнения объектов базы данных.
   
   - Откройте PL/SQL Developer.
   - Перейдите в меню **Tools** → **Compare User Objects**.
   - В окне сравнения выберите пользователя (схему), где находятся объекты, которые нужно сравнить.
   - Укажите объекты для сравнения (например, пакеты, процедуры).
   - Программа покажет различия между текущими и исходными версиями объектов базы данных.

2. **Сравнение SQL файлов с использованием внешних инструментов**: 
   
   - Если у вас есть исходный SQL файл (или пакет) и измененная версия, вы можете использовать сторонние программы для сравнения текстов файлов, например, **Beyond Compare**, **WinMerge** или **Meld**.
   - В PL/SQL Developer также можно загрузить свои SQL файлы в редактор и сравнить их вручную, или экспортировать объект в файл и затем провести сравнение.

3. **Версионирование кода**: 
   
   Если вы используете систему контроля версий (например, **Git**, **SVN** или **Mercurial**), рекомендуется сохранять изменения в репозитории. Это позволяет легко видеть историю изменений и сравнивать текущую версию с предыдущими.

4. **Сравнение версий объектов через Oracle Dictionary**: 
   
   - Если вы хотите сравнить объекты непосредственно в базе данных, можете создать скрипт для выборки информации из `DBA_SOURCE` или `USER_SOURCE`. Эти представления содержат исходный код всех процедур, функций и пакетов.
   - Вы можете экспортировать код объектов через SQL-запрос и сохранить их для последующего сравнения.
