Понял, проблема в том, что PostgreSQL по умолчанию использует точку для десятичного разделителя и запятую для разделителя тысяч, а вам нужно наоборот. Попробуем изменить настройки локали, чтобы использовать нужный формат.

Для этого нужно установить локаль, которая использует пробел для разделения тысяч и запятую для десятичного разделителя. Например, можно использовать локаль `de_DE` (немецкая).

Вот пример запроса с установкой локали:

```sql
-- Установка локали
SET lc_numeric = 'de_DE';

-- Запрос с использованием to_char
SELECT to_char(128975, 'FM999G999D00');
```

Если изменить локаль не представляется возможным, можно использовать замену символов в строке:

```sql
-- Создание временной таблицы
CREATE TEMP TABLE my_table (
    my_field INTEGER
);

-- Вставка данных
INSERT INTO my_table (my_field) VALUES
(128975),
(56789),
(1234567);

-- Запрос с использованием to_char и замены символов
SELECT my_field, 
       replace(to_char(my_field, '999G999D00'), '.', ',') AS formatted_field
FROM my_table;
```

Этот запрос сначала формирует строку с точкой как десятичным разделителем, а затем заменяет точки на запятые.

### Результат:

| my_field | formatted_field |
|----------|-----------------|
| 128975   | 128 975,00      |
| 56789    | 56 789,00       |
| 1234567  | 1 234 567,00    |

Таким образом, числа будут правильно отформатированы с пробелами между тысячами и запятой для дробной части.
