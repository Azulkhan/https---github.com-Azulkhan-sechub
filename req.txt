Похоже, что символ точки в значении `alias` может приводить к проблемам при построении URL. Возможно, значение `alias` используется в URL без должного кодирования, что может вызывать проблемы.

Чтобы правильно закодировать значение `alias` для использования в URL, нужно использовать функцию `encodeURIComponent`. Это гарантирует, что специальные символы, такие как точка, будут правильно обработаны в URL.

Попробуйте использовать следующий подход:

1. Закодируйте значение `alias` перед его использованием в URL.
2. Убедитесь, что значение декодируется при необходимости.

Пример:

```javascript
const aliasValue = 'm.sap_assetn';
const encodedAliasValue = encodeURIComponent(aliasValue);

this.fields[query_code].push({
       title: 'Номер ОС',
       alias: encodedAliasValue, 
       is_changed: 0, 
       _def_sel_query_code: null, 
       _data_type_code: 'varchar'
});
```

При построении URL:

```javascript
const url = `http://example.com/api?alias=${encodedAliasValue}`;
```

Если нужно декодировать значение обратно:

```javascript
const decodedAliasValue = decodeURIComponent(encodedAliasValue);
```

Этот подход должен помочь избежать проблем с исчезновением значения `alias` при добавлении в URL.
