
Если после установки Java версии 1.8 команда `java -version` всё ещё показывает версию 1.3, возможны следующие причины и способы их решения:

### 1. Версия Java в переменной среды PATH

Операционная система может продолжать использовать старую версию Java из-за настроек переменной среды `PATH`. Убедитесь, что путь к новой версии Java (в частности, к её исполняемым файлам в подпапке `bin`) находится в переменной среды `PATH` и что он указан до пути к старой версии.

- **Windows:** Измените переменную среды через "Системные свойства" -> "Дополнительные параметры системы" -> "Переменные среды".
- **macOS и Linux:** Добавьте или отредактируйте строки в файле `.bashrc`, `.bash_profile` или `.profile`, чтобы обновить переменную `PATH`.

### 2. Проверка и изменение переменной JAVA_HOME

Если вы используете переменную среды `JAVA_HOME` (что типично для некоторых приложений и серверов приложений), убедитесь, что она указывает на каталог новой установки Java 8.

### 3. Версии Java для разных приложений

Некоторые приложения могут явно указывать на использование определённой версии Java в своих конфигурационных файлах или ярлыках запуска. Проверьте настройки этих приложений, если они продолжают использовать старую версию Java.

### 4. Множественные установки Java

На вашей системе может быть установлено несколько версий Java. В некоторых случаях, команда `update-alternatives` (Linux) или подобные механизмы в других операционных системах позволяют выбрать, какая версия Java будет использоваться по умолчанию.

- **Linux:**
  Вы можете использовать команду `sudo update-alternatives --config java`, чтобы выбрать нужную версию.

### 5. Перезагрузка

После изменения переменных среды или установки новой версии Java может потребоваться перезагрузка вашего компьютера, чтобы изменения вступили в силу.

### 6. Удаление старой версии Java

В крайних случаях, если старая версия Java по-прежнему вызывается, вы можете попробовать удалить старую версию Java из системы, но перед этим убедитесь, что это не повлияет на работу необходимых приложений.

После выполнения этих шагов ещё раз проверьте версию Java командой `java -version`. Если все сделано правильно, должна отобразиться версия 1.8.
