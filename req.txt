Понял вас. Если вам нужно присоединить к пользователям только одну запись из таблицы `orders`, и выбрать именно ту, которая имеет самое большое значение `id` (т.е. последнюю запись), то можно использовать следующий подход с использованием подзапроса и `LEFT JOIN`.

### Пример:

Предположим, у вас есть две таблицы:

- `users` (основная таблица)
- `orders` (таблица с заказами, где может быть несколько записей для каждого пользователя)

Вы хотите выбрать каждого пользователя и присоединить только одну (последнюю) запись из таблицы `orders`, где у каждого `user_id` может быть несколько заказов.

### Решение с использованием подзапроса:

```sql
SELECT 
    u.*,
    o.*
FROM 
    users u
LEFT JOIN 
    orders o ON o.id = (
        SELECT id
        FROM orders
        WHERE orders.user_id = u.id
        ORDER BY id DESC
        LIMIT 1
    );
```

### Объяснение:

1. **Основной запрос:**
   - Выбираем всех пользователей из таблицы `users` (`u`).

2. **`LEFT JOIN` с подзапросом:**
   - Присоединение таблицы `orders` (`o`) происходит по полю `id`.
   - Подзапрос для каждого пользователя (`u.id`) выполняет выборку записи из таблицы `orders`, соответствующей `user_id = u.id`.
   - `ORDER BY id DESC` в подзапросе сортирует записи по `id` в порядке убывания (от последней к первой).
   - `LIMIT 1` ограничивает выборку одной записью — это будет последняя запись для данного `user_id`.

### Результат:
В результате вы получите список пользователей, и для каждого пользователя будет присоединена только одна запись из таблицы `orders` — последняя по `id`. Если у пользователя нет записей в `orders`, то данные о заказах будут `NULL` (из-за `LEFT JOIN`).
