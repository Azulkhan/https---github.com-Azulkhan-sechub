Чтобы изменить задание (job) в Oracle, если у пользователя, например, Светы, недостаточно привилегий, нужно выполнить несколько шагов:

1. **Предоставить необходимые привилегии**:
   Чтобы управлять заданиями, Свете необходимо обладать определёнными привилегиями. Это могут быть:
   - `CREATE JOB`
   - `MANAGE SCHEDULER`
   
   Пример запроса для предоставления привилегий:
   ```sql
   GRANT CREATE JOB TO sveta;
   GRANT MANAGE SCHEDULER TO sveta;
   ```

2. **Проверить привилегии**:
   Если привилегии предоставлены, но задание всё равно не изменяется, убедитесь, что у пользователя есть доступ к самому заданию, которое она пытается изменить.

3. **Выполнить команду через пользователя с достаточными привилегиями**:
   Если у Светы нет возможности получить привилегии, задание можно изменить от имени пользователя с более высокими правами (например, администратора), который обладает привилегиями `DBA`.

   Пример изменения задания:
   ```sql
   BEGIN
      DBMS_SCHEDULER.SET_ATTRIBUTE (
         name => 'sveta.my_job',
         attribute => 'job_action',
         value => 'new_action'
      );
   END;
   ```

4. **Использование роли**:
   Иногда проще дать временную роль с нужными привилегиями, если прямой доступ предоставлять нецелесообразно:
   ```sql
   GRANT DBA TO sveta;
   ```

После изменения задания не забудьте отозвать привилегии или роль, если это временная мера.
