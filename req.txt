Чтобы отслеживать HTTP-запросы и все, что происходит на веб-сервере в CentOS, можно использовать несколько подходов, в зависимости от того, какой веб-сервер вы используете (Apache, Nginx и т. д.). 

### 1. **Apache (httpd):**
Если у вас установлен Apache, журналы запросов хранятся в `/var/log/httpd/`.

- **Журнал доступа (access log):** здесь хранятся все запросы к серверу.
  ```bash
  tail -f /var/log/httpd/access_log
  ```

- **Журнал ошибок (error log):** здесь содержится информация о любых ошибках.
  ```bash
  tail -f /var/log/httpd/error_log
  ```

Команда `tail -f` выводит последние строки файла и автоматически обновляет вывод, когда появляются новые записи.

### 2. **Nginx:**
Для Nginx журналы находятся в `/var/log/nginx/`.

- **Журнал доступа (access log):**
  ```bash
  tail -f /var/log/nginx/access.log
  ```

- **Журнал ошибок (error log):**
  ```bash
  tail -f /var/log/nginx/error.log
  ```

### 3. **Использование `tcpdump` для мониторинга HTTP-трафика:**
Если вы хотите отслеживать все HTTP-запросы на уровне сети, можно использовать `tcpdump`. Пример команды для захвата трафика на порту 80 (HTTP):
```bash
sudo tcpdump -i any -n port 80
```

Это захватит весь HTTP-трафик на порту 80. Для HTTPS (порт 443):
```bash
sudo tcpdump -i any -n port 443
```

### 4. **Использование `ngrep` для фильтрации HTTP-запросов:**
`ngrep` (Network grep) — это утилита, которая позволяет фильтровать HTTP-запросы по ключевым словам.

Чтобы установить `ngrep`, выполните:
```bash
sudo yum install ngrep
```

Пример команды для просмотра всех HTTP-запросов:
```bash
sudo ngrep -d any -W byline 'GET|POST' tcp port 80
```

### 5. **Журналы приложений:**
Если у вас веб-приложение, которое пишет собственные логи, эти файлы журналов можно найти в его каталоге или в системных логах (например, `/var/log`).
