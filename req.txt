В CentOS, для извлечения информации из сертификата `.cer` (или `.crt`) файла с использованием OpenSSL и записи этой информации в текстовый файл, можно использовать следующий сценарий:

1. Убедитесь, что OpenSSL установлен. Если нет, установите его командой:
    ```sh
    sudo yum install openssl
    ```

2. Извлеките информацию из сертификата и запишите её в текстовый файл.

Предположим, у вас есть сертификат `certificate.cer`. Чтобы извлечь информацию, используйте следующую команду:
```sh
openssl x509 -in certificate.cer -noout -subject
```

Эта команда выведет строку с информацией о субъекте, включая ФИО, если оно присутствует в сертификате.

3. Чтобы записать эту информацию в текстовый файл, можно перенаправить вывод команды в файл:
```sh
openssl x509 -in certificate.cer -noout -subject > subject_info.txt
```

Если вам нужно извлечь только ФИО (если оно указано в формате `CN`), то можно использовать команду `grep` и `cut` для фильтрации:
```sh
openssl x509 -in certificate.cer -noout -subject | grep 'CN=' | cut -d'=' -f2 > fio.txt
```

Этот сценарий предположительно будет работать, если `CN` (Common Name) содержит ФИО субъекта. Команда `cut -d'=' -f2` извлекает значение, которое следует после `CN=`.

Вот пример полного сценария:
```sh
#!/bin/bash

# Убедитесь, что файл сертификата указан
if [ -z "$1" ]; then
    echo "Usage: $0 <certificate.cer>"
    exit 1
fi

# Имя файла сертификата
CERT_FILE="$1"

# Извлечение информации о субъекте
SUBJECT=$(openssl x509 -in "$CERT_FILE" -noout -subject)

# Извлечение ФИО (если находится в CN)
FIO=$(echo "$SUBJECT" | grep 'CN=' | cut -d'=' -f2)

# Запись ФИО в файл
echo "$FIO" > fio.txt

echo "ФИО субъекта записано в fio.txt"
```

Сохраните этот сценарий в файл, например `extract_fio.sh`, сделайте его исполняемым и запустите:
```sh
chmod +x extract_fio.sh
./extract_fio.sh certificate.cer
```

Этот скрипт извлечёт информацию о субъекте сертификата и запишет ФИО в файл `fio.txt`.
