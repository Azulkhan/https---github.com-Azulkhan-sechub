Для того чтобы проверить, какие права есть у конкретного пользователя в Linux, можно использовать несколько команд для анализа прав на файлы, групп и возможностей sudo. Вот несколько способов:

### 1. **Проверка групп пользователя**:
   Для проверки групп, к которым принадлежит пользователь, используйте команду `groups`. Это покажет, к каким группам принадлежит текущий или указанный пользователь, что может определять его права на системе.

   - Для текущего пользователя:
     ```bash
     groups
     ```

   - Для конкретного пользователя:
     ```bash
     groups <имя_пользователя>
     ```

   Также можно использовать команду `id`, которая даст больше информации, включая UID и GID:
   ```bash
   id <имя_пользователя>
   ```

### 2. **Проверка прав на выполнение команд через `sudo`**:
   Чтобы узнать, какие права у пользователя с помощью `sudo`, выполните команду:

   - Для просмотра всех привилегий:
     ```bash
     sudo -l
     ```

   Эта команда покажет, какие команды пользователь может выполнять с повышенными правами через `sudo`.

### 3. **Проверка прав на файлы**:
   Для того чтобы узнать, какие права пользователь имеет на конкретный файл или директорию, используйте команду `ls -l`. Она покажет разрешения для владельца, группы и других пользователей.

   Пример команды:
   ```bash
   ls -l /путь/к/файлу
   ```

   Пример вывода:
   ```
   -rwxr-xr-- 1 user group 4096 Oct 18 12:34 файл.txt
   ```

   Здесь важны первые символы:
   - Первый символ показывает тип объекта (например, `-` для файла или `d` для директории).
   - Следующие три символа показывают права владельца (чтение, запись, выполнение).
   - Далее идут три символа для группы.
   - Последние три символа — права для всех остальных.

### 4. **Проверка прав с помощью `getfacl`**:
   В случае, если на вашем сервере используется система доступа ACL (Access Control List), можно проверить права на файл или директорию с помощью `getfacl`.

   Для установки (если не установлено):
   ```bash
   sudo apt install acl
   ```

   Для использования:
   ```bash
   getfacl /путь/к/файлу
   ```

### 5. **Просмотр пользователя в `/etc/passwd` и групп в `/etc/group`**:
   Вы можете вручную проверить права пользователя, просмотрев файлы конфигурации.

   - Для просмотра всех пользователей:
     ```bash
     cat /etc/passwd
     ```

   - Для просмотра групп:
     ```bash
     cat /etc/group
     ```

Эти шаги помогут определить, какие права есть у пользователя на системе и что он может выполнять.
